🐾 wolfie.love Shop — How It Works

Concept documentation · wolfie.love/shop

A product page that
reads like a story.

wolfie.love/shop applies the sentence-by-sentence reading model to product discovery. The Air Jordan 1 Retro High OG in Pale Ivory × Psychic Blue is the subject. Five chapters, one pair of shoes, a complete reading experience before anything is added to a cart. This document covers every mechanic — the sentence engine, chapter themes, the HUD, the image layer, the commerce panel, scroll mode, and the end-of-journey moment.

01

The Argument

Most product pages assume the reader already wants to buy. They skip directly to the decision — size grid, add to cart, proceed. The page is a transaction form with photographs attached.

wolfie.love/shop makes the opposite bet. The product earns the purchase. Five chapters, read sentence by sentence, move from cultural origin to material craft to technical specification to the urgency of the drop. Commerce is the destination, not the premise.

The format is the argument. Slowing the reader down is not a friction problem — it is the product's best case, made at the pace it deserves.

Every interaction decision follows from this. Sentences advance one at a time. The panel stays visible but passive. The HUD tells you where you are. The background shifts with each chapter. You finish the piece, and then you decide.

02

Five Chapters

The narrative is divided into five chapters, each a distinct emotional register for the same object. Not a product description split into sections — five different lenses on one shoe.

Chapter 01
Entry

The name. The colorway. Pure cinema dark. The shoe arrives before the story does.

Chapter 02
The Origin

1985. The $5,000 fine. The rule the NBA wrote and Jordan ignored. Cultural context as product context.

Chapter 03
The Craft

Textured Psychic Blue suede. Visible seams. Six lace jewels as a personality test. The details you can see in the images.

Chapter 04
The Details

Style number, colorway, materials, lining, retail. The spec sheet, made to breathe.

Chapter 05
The Moment

The drop. The window measured in minutes, not days. Nike Members first. The piece ends at the decision.

Each chapter has its own centered header — number, title, subhead — that fades in when the reader reaches it. Chapter headers are not navigational. They are structural markers that appear at the right moment, not on demand.

03

Sentence Engine

Every element in the narrative stream — poster type, body copy, spec rows, labels, breaks — is a discrete DOM node with its own state. The engine advances through these nodes one at a time and manages three visibility states across the full stream.

Active

Full opacity. The current sentence. Poster sentences display a thin left-edge accent rule in the chapter's accent color. Body sentences read at full ink weight. Spec rows illuminate their value text.

Revealed

Already read. Dimmed to approximately 38% opacity — present as context, not demanding attention. The reader knows where they've been without being pulled backward.

Unrevealed

Not yet reached. Dimmed to 12% — present but nearly invisible. The structure of what's ahead is legible; the content isn't. There is always more.

Navigation is triggered by scroll wheel, trackpad, arrow keys, j / k, Space, swipe (mobile), or the HUD prev/next buttons. All input methods work simultaneously. The active sentence is always scrolled into the center of the viewport automatically.

Sentence Types

Poster — giant display type. Product name, colorway. The emotional arrival. Body — narrative prose at reading size. The story. Spec — monospace key/value pairs. The facts. Label — section markers in accent color. Break — visual breath between sections.

Details Auto-Flow

The Details chapter (Chapter 04) uses an auto-flow flag on each spec row. Once the reader reaches the first spec, the engine advances automatically through the remaining rows at 950ms intervals — the spec sheet scans itself. A thin accent-color scan line animates across each active spec row as the timer counts down. Any manual navigation cancels the timer and returns control immediately.

Clicking any sentence in the stream jumps directly to it. The engine re-evaluates all revealed/active/unrevealed states from the new position.

04

Chapter Themes

Each chapter carries a complete visual identity — a different palette, a different mood, a different color relationship with the shoe. The theme shifts automatically as the reader advances into each chapter. Transitions take 1.2 seconds and affect every themed element simultaneously.

Ch. 01 · Entry

Ch. 02 · The Origin

Archival. Historical. Warm paper.

Ch. 03 · The Craft

Chapter 05 (The Moment) returns to an electric blue — darker than Ch. 03, more urgent. The palette signals arrival at the decision point before the copy does.

HUD Inversion

The navigation HUD uses an inverted color scheme — light surface, dark text — that is independent of the chapter theme. Each chapter defines its own --hud-* variable set: warm cream for Ch. 02, cool periwinkle for Ch. 03, pale ivory for Ch. 04. The HUD always reads cleanly against the dark editorial background regardless of which chapter theme is active.

Transition Scope

Theme transitions apply to: the background color, the image overlay gradient, the accent color throughout the stream, the HUD shell and values, the panel background on hover, the chapter header accent rule, and all spec/label elements. The transition is choreographed — everything moves together at 1.2s ease.

05

Image Layer

Six product photographs occupy a fixed layer behind the entire page — not inline in the content flow, not thumbnails, not a carousel. Full viewport. Behind the prose. Surfaced by which sentence is active.

Sentence-Bound Images

Each sentence carries a data-img attribute pointing to one of the six images. When a sentence becomes active, its bound image cross-fades into view behind the content. The image changes with the sentence — the photographs advance with the reading.

The Six Views

Top-down pair (entry). Alternate top angle (the story). Heel and outsole detail (the craft). Insoles with heritage print (the details). Lace jewels frontal (the craft). Full pair (the moment). Each image is matched to the chapter content it supports.

Opacity and Overlay

Images display at 55% opacity — bright enough to read, dark enough to read through. A directional gradient darkens the left side (where text lives) from 82% to 45%, leaving the right edge of the viewport nearly full exposure. The overlay transitions with chapter themes, shifting warmth and depth as the palette changes.

Ken Burns

Each newly active image begins at scale(1.03) and eases to scale(1.0) over 6 seconds — a slow, imperceptible drift. The photographs breathe. They are not static assets.

06

The HUD

The navigation HUD is a fixed vertical pill on desktop — light surface, dark ink, independent of the chapter theme. It sits at the right edge of the content column, 400px from the right viewport edge (just outside the commerce panel). It never changes size or shape. It is always in the same place.

Controls

Top to bottom: a drag handle (three dots), a divider, three action buttons (Views, Panel, Notify), a divider, then the navigation cluster. The navigation cluster is: ↑ previous, a progress pip (vertical bar filling top-to-bottom as the reader advances), ↓ next. The pip also shows the sentence counter below it.

Action Buttons

Views — opens the full image grid overlay. All six product photographs on a black canvas, zoomable, pannable. Panel — scrolls the commerce panel into view and pulses its border briefly. Notify — appears near the end of the journey when the reader is close to finishing. Triggers the notification toast.

Draggable

The HUD can be repositioned anywhere on screen by dragging the handle. On first drag it converts from CSS right/top anchoring to absolute left/top positioning, preserving its visual location. Position holds for the session.

Panel Hover Fade

On desktop, hovering the commerce panel fades both the HUD and the chapter widget to opacity 0 with pointer-events disabled. The panel is the only thing that matters in that moment. Mousing away restores both at 0.4s ease.

Scroll Mode Fade

When scroll mode is active, the HUD fades to 18% opacity and becomes non-interactive. It signals that navigation has been handed off to native scroll, and waits.

07

Hotspot Zones

Every navigation paradigm on the web requires you to find a control and aim at it. Buttons, arrows, scroll bars — all demand precision targeting. The hotspot zone system inverts this entirely. The content column itself is the control surface. Where your cursor is on screen is your intent.

The Five Zones

The content column is divided vertically by cursor position: top 10% jumps to the first sentence, 10–33% navigates to the previous sentence, 33–67% is a dead zone with no action, 67–90% navigates to the next sentence, and the bottom 10% jumps to the last sentence. No buttons. No labels. Just where you are.

The Dead Zone Is the Point

The middle third doing nothing is what makes the whole system work. If the entire column were active, every mouse movement through the reading area would feel loaded and anxious. The dead zone lets you rest — you can read, hover over sentences, exist in the space without triggering anything. The hotspots only fire when you reach toward the edges, when your body is already expressing directional intent through movement. Stillness is safe. Reaching means something.

The Cursor as Feedback

The custom cursor changes shape and label in each zone — ↑ for previous, ↓ for next, ⤒ for jump to first, ⤓ for jump to last. You never have to guess whether a zone registered. The cursor tells you immediately what the page thinks you're about to do. When you hover a sentence directly, all hotspot states clear — the sentence itself is the target, not the zone. The system yields to precision when precision is offered.

Spatial Intention Reading

The zones map to a spatial metaphor the body already knows: near the top means further back, near the bottom means further forward. This isn't learned behavior — it's borrowed from how people physically relate to vertical space. The interface catches up to an intuition that already exists. No instruction needed.

The hotspot zones make the entire content column navigational without making it feel like a navigation element. The reading space and the control surface are the same thing.

08

Chapter Widget

The chapter widget is a separate element from the HUD — a dark pill centered at the top of the content column, always visible, always dark regardless of chapter theme. It shows the current chapter name and opens a dropdown of all five chapters.

The Pill

Chapter · {name} ▾ — a horizontal row in rgba(10,10,12,0.88) with a thin white border at 10% opacity. Click opens the dropdown downward with a spring scale animation. The arrow rotates 180° when open.

The Dropdown

Lists all five chapters with their number and name. The current chapter shows full-brightness text and an accent dot on the right. Clicking any item jumps to the first sentence of that chapter — the engine calculates the correct sentence index at load time and stores it for instant lookup. The dropdown closes on item selection, outside click, or Escape.

Positioning

On desktop: position: fixed, spanning left: 0 to right: 380px (the content column width), with display: flex; justify-content: center — the pill floats at true center of the content column, 68px below the header. On mobile: spans full width below the header. Always dark. Always readable.

09

Commerce Panel

The commerce panel is a sticky right column — 380px wide on desktop, full-width below the narrative stream on mobile. It is always accessible. It holds everything needed to act on the piece: the product image, thumbnails, colorway picker, availability status, and the CTA.

Reading Progress

A thin horizontal bar at the top of the panel shows how far through the narrative the reader has progressed — not how far they've scrolled, but how many sentences they've activated. The label reads Read. Commerce is downstream of reading.

Product Image

A square image area synced with the active sentence. When the narrative moves to a new image, the panel image updates to match — the panel and the background layer show the same photograph simultaneously. Four thumbnail buttons below the image allow manual selection, decoupled from the narrative if the reader wants to look ahead.

Colorway Picker

Two circular swatches — Pale Ivory / Psychic Blue and Pale Ivory / Pro Green. Gradient-filled circles rendered in CSS. Active swatch scales up slightly and receives an accent border. Selecting a colorway updates the displayed name below.

Coming Soon State

The shoe is not yet available. The size grid is replaced by an availability block explaining that Nike Members get first access. The primary CTA is Notify Me on Nike.com — which triggers the toast rather than opening a new tab. The panel signals the commercial reality without pretending the shoe is available.

Panel Hover Theme

Hovering the panel flips it to a near-white surface with dark ink — a deliberate inversion from the editorial darkness of the left column. The panel becomes a bright, commercial object when approached. It knows what it is.

Image Grid — Views

The Views button opens a full-screen overlay containing all six product images rendered on a canvas. The images are pannable and zoomable. The grid is titled and labeled. Close via the close button or Escape key.

10

Scroll Mode

Scroll Mode is an escape hatch. It releases the sentence-by-sentence constraint and lets the page behave like a normal document — all sentences fully visible, native scrolling restored. It is not the intended reading experience. It exists for readers who won't make the trade.

The Toggle

A small circle in the top-right of the content column (desktop) or top-right below the header (mobile). Ghost treatment: rgba(255,255,255,0.04) fill, thin white border at 18% opacity. When active, it glows with the accent color and pulses gently — a reminder that you made a choice and can unmake it.

Entering Scroll Mode

All sentences reveal at full opacity. A lockout overlay sits over the stream content at z-490, absorbing pointer events — the sentence engine does not activate on scroll, but it does respond to a click anywhere in the content area. That click is the exit. The panel remains fully interactive — commerce is never locked. The HUD fades to 18% opacity.

Exiting Scroll Mode

A click anywhere in the stream — or a tap on the toggle itself — exits scroll mode. The lockout overlay is the exit trigger: it absorbs the pointer event, turns itself off, and hands the click back to the engine. The toggle extinguishes as a consequence, not a cause. The reader stays exactly where they are — goTo(current, { noScroll: true }) restores sentence states around the current scroll position without moving the viewport.

Mobile Tappability

The toggle carries pointer-events: auto !important unconditionally — it cannot be blocked by the lockout overlay regardless of z-index behavior on any platform. The toggle is always tappable. There is always a way out.

11

End of Journey

When the reader reaches the final navigable sentence, two things happen. The Notify button appears in the HUD. And the commerce panel pulses.

Panel Beckon

The panel fires a journey-end CSS animation — a glowing inset border in the chapter's accent color, with a soft halo, pulsing twice over 1.8 seconds. The animation runs once and removes itself cleanly. The panel is saying: now. The piece is finished. The decision is yours.

Notify CTA

Both the panel button and the HUD Notify button trigger the same toast. Not a redirect to Nike.com — a toast. The demo context makes the commitment. In production, this would hand off to the Nike notification flow.

The Toast

A spring-animated panel rising from the bottom of the viewport: 🐾 icon, You're on the list headline, Nike will notify you the moment it drops · wolfie.love subline, a dismiss button. Auto-dismisses after 5 seconds. The wolfie.love credit is in the subline because this is still wolfie.love, even here.

12

Mobile

Mobile is a first-class reading experience with its own layout and interaction model. The two-column desktop layout collapses to a single column — narrative stream above, commerce panel below. The HUD becomes a bottom bar. The chapter widget stays at top center.

Bottom HUD Bar

On mobile, the HUD is a full-width horizontal bar pinned to the bottom of the viewport. Four equal-flex slots: Views, Panel, ↑, ↓. No pip, no drag handle, no extra cells — only what a mobile reader actually reaches for. Rounded top corners (20px). Safe-area padded for iOS home indicator and Android gesture bar.

Swipe Navigation

Swipe up to advance, swipe down to retreat. A 40px threshold distinguishes swipe from tap. A cooldown prevents double-advances on gesture momentum. All sentence navigation works normally via swipe.

Chapter Widget on Mobile

The chapter widget spans the full viewport width on mobile (no panel to account for), centered at 10px below the 52px header. Same dark pill, same dropdown behavior, same dark treatment. The dropdown opens downward from the pill.

Scroll Toggle on Mobile

Fixed at top-right below the header (top: 64px; right: 16px). Ghost treatment: near-transparent fill, thin white border at 18% opacity. Slightly smaller than desktop (44px vs 52px). When active, glows accent and stays in place rather than floating to a fixed corner.

Commerce Panel

Below the stream, full width, static position. All panel interactions — thumbnail selection, colorway picker, CTA — work identically to desktop. The reading progress bar remains at the top of the panel on mobile.

13

Technical Notes

Built from scratch. No frameworks. No build step. No dependencies beyond Google Fonts.

System Implementation
Sentence Engine Custom state machine over DOM nodes with data-idx, data-ch, data-img attributes. Three states: unrevealed / revealed / active. goTo(n) accepts an optional { noScroll } flag.
Chapter Themes CSS custom properties on :root, overridden by body.ch-N classes. Separate --hud-* variable set for the inverted HUD palette. Transitions at 1.2s ease across all themed elements simultaneously.
Image Layer Fixed-position element, z-index 0, behind everything. Six <img> elements, opacity toggled via .visible class at 0.55 opacity. Ken Burns via scale(1.03) → scale(1.0) over 6s. Overlay gradient in ::after pseudo-element.
HUD Fixed vertical pill at right: 400px; top: 50%; transform: translateY(-50%). Drag converts to absolute positioning on first move. Progress pip is a vertical bar using height: pct% transitions.
Chapter Widget Fixed container spanning content column (left:0; right:380px), flexbox centered. Dropdown uses CSS scaleY transform from transform-origin: top center. Chapter-first-sentence index map built at load time from the navigable array.
Auto-Flow setTimeout chain at 950ms for .auto-flow spec rows. Cleared on any manual navigation input. Scan-line animation via ::after pseudo-element with a @keyframes width sweep.
Panel Beckon journey-end CSS animation class added on last sentence activation. Force-reflow via void panel.offsetWidth to restart if triggered twice. animationend listener removes the class once.
Scroll Lockout Fixed overlay at z-490 absorbing pointer events over the stream. Toggle at z-501 with pointer-events: auto !important unconditionally — cannot be blocked. Exit calls goTo(current, { noScroll: true }).
Grain Texture Inline SVG feTurbulence filter on a fixed body::after pseudo-element at 6% opacity, z-index 999, pointer-events none.
Image Grid Full-screen canvas overlay. Six images rendered via the Canvas 2D API, pannable and zoomable via mouse drag and wheel events. Opens at z-400.
Fonts Barlow Condensed (display, poster type) · Space Mono (UI, specs, labels). Google Fonts via preconnect.

The panel hover fade uses body:has(#panel:hover) #nav-widget — a single CSS rule that reaches across the DOM to fade the HUD whenever the commerce panel is in use, regardless of where the user dragged it.
